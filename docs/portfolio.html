■ 프로젝트 개요

핵심 목표

비개발자(QA 담당자)가 코드 작성 없이 시각적 플로우차트 인터페이스만으로 모바일 게임 자동화 테스트 시나리오를 생성하고, 다중 디바이스에서 병렬로 실행할 수 있는 통합 QA 자동화 플랫폼

  • [v] 시각적 시나리오 편집기 - 드래그 앤 드롭으로 테스트 플로우 구성
  • [v] 이미지 인식 기반 액션 - OpenCV 템플릿 매칭으로 UI 요소 인식
  • [v] 다중 디바이스 병렬 실행 - 최대 10대 디바이스 동시 테스트
  • [v] 실시간 모니터링 - Socket.IO 기반 진행 상황 실시간 확인
  • [v] 상세 리포트 - 스크린샷, 비디오 녹화, 성능 메트릭 포함
  • [v] 스케줄링 - Cron 기반 자동 테스트 예약 실행

■ 서브 프로젝트 구성

1. Backend Server
시나리오 관리, 디바이스 제어, 테스트 실행 엔진, 리포트 생성을 담당하는 핵심 서버. Appium을 통해 Android 디바이스와 통신하며, Socket.IO로 실시간 상태를 전송.
Node.js 22 Express 5 TypeScript Socket.IO Appium sharp OpenCV
2. Frontend App
시각적 플로우차트 편집기, 실시간 디바이스 미리보기, 테스트 실행 패널, 리포트 뷰어를 제공하는 웹 애플리케이션. Context API로 상태 관리, 가상 스크롤로 대량 데이터 처리.
React 19 TypeScript Vite Socket.IO Client react-window
3. Server Manager
Backend, Frontend, Appium 서버를 통합 관리하는 데스크톱 애플리케이션. 한 번의 클릭으로 전체 시스템 시작/종료, 실시간 로그 모니터링 제공.
Electron 28 React TypeScript Vite
4. QA Recorder App
테스트 대상 디바이스에 설치되는 Android 앱. 화면 녹화, 스크린샷 캡처, OpenCV 템플릿 매칭을 ADB 브로드캐스트 명령으로 수행.
Android Kotlin MediaProjection OpenCV

■ 핵심 기능

1. 시각적 시나리오 편집기

Canvas 기반 플로우차트
노드 추가/삭제/이동, 연결선 그리기, 줌 제어, 다중 선택, 복사/붙여넣기 등 직관적인 플로우차트 편집 기능 제공
액션 파라미터 패널
터치, 스와이프, 텍스트 입력, 이미지 인식 등 다양한 액션의 파라미터를 디바이스 미리보기와 연동하여 설정

2. 이미지 인식 시스템

OpenCV 템플릿 매칭을 활용하여 화면의 UI 요소를 인식하고, 해당 위치에 터치/클릭 액션 수행

  • 멀티스케일 매칭 - 다양한 해상도에서 동작
  • ROI(관심 영역) 설정 - 검색 범위 제한으로 성능 향상
  • 임계값 조정 - 매칭 정확도 제어
  • 하이라이트 표시 - 매칭 결과 시각화

3. 다중 디바이스 병렬 실행

기능 설명
독립 세션 관리 각 디바이스마다 별도의 Appium 세션 생성, MJPEG 포트 자동 할당
병렬 실행 Promise.allSettled로 디바이스 독립 실행 (한 디바이스 실패해도 다른 디바이스 계속)
실시간 상태 동기화 Socket.IO로 모든 클라이언트에 진행 상황 브로드캐스트
디바이스 큐 사용 중인 디바이스에 테스트 요청 시 대기열 처리

4. 상세 테스트 리포트

  • [v] 단계별 실행 결과 - 각 노드의 성공/실패 여부, 소요 시간
  • [v] 스크린샷 갤러리 - 실행 중 캡처한 스크린샷 (가상 스크롤로 대량 처리)
  • [v] 비디오 녹화 - 전체 테스트 과정 녹화 + 타임라인 마커
  • [v] 성능 메트릭 - 각 액션의 실행 시간, 평균/최대/최소 통계
  • [v] 실패 분석 - 에러 메시지, 스택 트레이스, 실패 시점 스크린샷

■ 기술적 특징

모듈화된 실행 엔진
testExecutor를 6개 모듈로 분리하여 단일 책임 원칙 준수

- ActionExecutionService: 액션 실행 + 재시도
- NodeNavigationService: 흐름 제어 (조건/루프)
- ExecutionStateManager: 상태 관리
- ExecutionMediaManager: 미디어 캡처
- ScenarioExecutionEngine: 시나리오 실행
- PerformanceMetricsCollector: 메트릭 수집
Context 기반 상태 관리
AppStateContext를 4개 Context로 분리

- FlowEditorContext: 노드/연결 편집
- ScenarioEditorContext: 시나리오 관리
- EditorPreviewContext: 미리보기 연동
- UIContext: UI 상태 (탭, 모달)
실시간 통신
Socket.IO 양방향 통신으로 실시간 상태 동기화

- 테스트 진행 상황 (노드별 상태)
- 디바이스 큐 상태 업데이트
- 다중 사용자 동시 접속 지원
성능 최적화
10대 디바이스 동시 지원을 위한 최적화

- 폴링 주기 최적화 (10초/3초)
- 가상 스크롤 (react-window)
- MJPEG 스트리밍 (HTTP 기반)
- Rate Limiting (요청 제한)
타입 안정성
100% TypeScript 적용

- Zod 스키마로 API 요청 검증
- 공유 타입 정의 (types/)
- tsc --noEmit 타입 체크
에러 처리
계층별 에러 처리 전략

- ActionExecutionService: 재시도 로직
- SessionCrashError: 세션 복구
- ErrorBoundary: React 에러 폴백
- validateSchema: 입력 검증

카테고리 액션 설명

swipe 시작점에서 끝점으로 스와이프
longPress 길게 누르기

앱 activateApp 앱 활성화 (포그라운드)

■ 주요 API

엔드포인트 메서드 기능
/api/scenarios GET / POST 시나리오 목록 조회 / 생성
/api/device/list GET 연결된 디바이스 목록
/api/session/create POST Appium 세션 생성
/api/test/execute POST 테스트 실행 (병렬)
/api/test-reports GET 리포트 목록 조회
/api/image/match POST 이미지 템플릿 매칭
/api/schedules GET / POST 스케줄 관리
